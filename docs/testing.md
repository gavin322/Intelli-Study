# 测试与验证计划

## 单元测试
- 后端：使用 Jest，对认证、词库、练习服务进行单元测试，目标覆盖率 ≥70%。
- 前端：使用 Vitest + Vue Test Utils，对关键组件（学习卡片、练习表单等）进行渲染与交互测试。

## E2E 自动化
- 后端：`npm run test:e2e` 使用 Supertest 走通健康检查，后续扩展注册→练习→统计流程。
- 前端：预留 Playwright 目录，未来补充“注册→学习→练习→错词统计”全链路场景。

## 手工验收步骤
1. 注册新用户并登录，确认个人中心显示邮箱。
2. 进入学习页，滑动卡片并标记掌握/生词本。
3. 进入默写页，故意拼错 2 个单词，查看提示，确认错误次数统计。
4. 打开学习历史，确认错词列表展示累计错误次数。
5. 使用手机浏览器访问 `/learn`，确认卡片可滑动并能操作按钮。

## 数据准备
- 可通过 Prisma Seed（后续补充）或 SQL 脚本导入词库。
- docker-compose 默认创建 `zhixie` 数据库，可直接用于本地联调。
